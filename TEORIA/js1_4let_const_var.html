1.4 — Variabili: let, const, var

In Python una variabile si crea semplicemente assegnandole un valore: x = 5.
In JavaScript esistono tre modi per dichiarare una variabile, e la scelta conta.

1. let — variabile che può cambiare

let è il modo moderno per dichiarare una variabile il cui valore potrà essere riassegnato in seguito. 

// Dichiarazione + assegnazione in un'unica riga
let nome = "Mario";
console.log(nome);    // → "Mario"

// Riassegnazione: ✅ consentita con let
nome = "Luigi";
console.log(nome);    // → "Luigi"

// Posso anche dichiarare la variabile prima e assegnarle un valore dopo
let eta;              // eta vale undefined (non ancora assegnata)
eta = 25;
console.log(eta);     // → 25

2. const — costante (non riassegnabile)

const crea un "legame" fisso tra il nome e il valore: non puoi riassegnare la variabile dopo averla dichiarata. 

const PI = 3.14159;
console.log(PI);      // → 3.14159

PI = 3;               // ❌ TypeError: Assignment to constant variable
                      // Il browser lancia un errore e blocca l'esecuzione

Attenzione: const ≠ immutabile

const blocca la riassegnazione (non puoi puntare a un nuovo valore), ma non impedisce la modifica del contenuto interno di oggetti e array. 

const numeri = [1, 2, 3];

numeri.push(4);            // ✅ modifica il contenuto dell'array → OK
console.log(numeri);       // → [1, 2, 3, 4]

numeri = [5, 6, 7];        // ❌ riassegnazione → TypeError!
// numeri puntava a [1,2,3,4], non posso farlo puntare a un nuovo array
💡 Analogia: immagina const come un cartello con il nome scritto sopra. Il cartello è incollato a una scatola: non puoi spostarlo su un'altra scatola, ma puoi tranquillamente aggiungere oggetti dentro la scatola.

3. var — il vecchio modo (da evitare)

var è la keyword originale di JavaScript (pre-2015). Funziona, ma ha comportamenti controintuitivi legati allo scope (l'ambito di visibilità della variabile) che vedremo nella Lezione 3. 

var eta = 20;          // funziona, ma...

// PROBLEMA 1 — var "esce" dai blocchi if/for (non da let/const)
if (true) {
    var dentro = "visibile fuori!";
    let fuori  = "rimane dentro";
}
console.log(dentro);   // → "visibile fuori!" (comportamento di var)
console.log(fuori);    // ❌ ReferenceError (let rispetta il blocco)

// PROBLEMA 2 — var permette di ridichiarare la stessa variabile senza errori
var x = 10;
var x = 20;            // nessun errore... ma è quasi sempre un bug silenzioso

// con let questo è un errore esplicito e quindi più sicuro:
let y = 10;
let y = 20;            // ❌ SyntaxError: Identifier 'y' has already been declared
📌 Perché studiare var se è da evitare?
Perché troverai var in tantissimo codice legacy (vecchio) su GitHub, Stack Overflow e tutorial datati. Devi riconoscerlo e capirlo.
Regola d'oro:

✅ Usa sempre const di default.
✅ Usa let solo quando sai che il valore dovrà cambiare.
❌ var è un'eredità del passato — non usarla in codice nuovo.

4. Confronto con Python

Python JavaScript
Variabile x = 5 let x = 5;
Costante X = 5 (solo convenzione) const X = 5; (enforced!)
Riassegnazione sempre possibile let sì, const no
Punto e virgola non necessario consigliato ;
Tipo dichiarato no (dinamico) no (dinamico, TypeScript lo aggiunge)
💡 Punto e virgola: In JS è tecnicamente opzionale grazie all'Automatic Semicolon Insertion (ASI), ma è buona pratica metterlo sempre. Evita sorprese in casi limite come: 
// Senza punto e virgola, JS può interpretare male il codice:
const a = 1
const b = 2
[a, b].forEach(console.log)   // TypeError — JS legge: const b = 2[a,b]...

// Con punto e virgola nessun problema:
const a = 1;
const b = 2;
[a, b].forEach(console.log);  // ✅ stampa 1 e 2
