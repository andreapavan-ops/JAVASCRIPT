1.8 — Operatore ternario e switch

Due strumenti per scrivere logica condizionale in modo compatto. L'operatore ternario esiste anche in Python (con sintassi diversa); lo switch invece non ha equivalente diretto in Python (fino a Python 3.10).

1. Operatore ternario ? :

È un if/else su una sola riga che restituisce un valore. Sintassi: 

// condizione ? valore_se_vera : valore_se_falsa
let risultato = (condizione) ? "sì" : "no";
Confronto con Python: 
// JavaScript
let messaggio = (eta >= 18) ? "Maggiorenne" : "Minorenne";

// Python — stessa logica, sintassi diversa
// messaggio = "Maggiorenne" if eta >= 18 else "Minorenne"
Regola #1 — Il ternario è un'espressione, non uno statement.
Restituisce sempre un valore, quindi può essere usato direttamente in un'assegnazione, in un console.log, in un template literal o come argomento di una funzione. Non usarlo per eseguire effetti collaterali (come chiamate a funzioni con side-effect): per quelli usa un normale if/else. 
// ── Uso base ──────────────────────────────────────────────────────────
let eta = 20;
let accesso = (eta >= 18) ? "Accesso consentito" : "Accesso negato";
console.log(accesso);                    // "Accesso consentito"

// ── Direttamente nel console.log (REGOLA #1) ──────────────────────────
console.log(eta >= 18 ? "adulto" : "minore");   // "adulto"

// ── In un template literal ─────────────────────────────────────────────
let nome = "Anna";
let punteggio = 85;
console.log(`${nome} è ${punteggio >= 60 ? "promossa" : "bocciata"}.`);
// "Anna è promossa."

// ── Come argomento ─────────────────────────────────────────────────────
let numeri = [5, -3, 8, -1, 0];
numeri.forEach(n => {
    console.log(n, n > 0 ? "positivo" : n < 0 ? "negativo" : "zero");
});
// 5  "positivo"
// -3 "negativo"
// 8  "positivo"
// -1 "negativo"
// 0  "zero"

1.1 Ternari annidati

È possibile annidare ternari (un : che contiene un altro ? :), ma oltre due livelli il codice diventa illeggibile.

Regola #2 — Non annidare più di due ternari.
Se hai tre o più condizioni, usa if/else if/else o switch: il guadagno in brevità non vale la perdita di leggibilità. 
// ✅ Due livelli: accettabile
let voto = 7;
let giudizio = voto >= 9 ? "ottimo"
             : voto >= 6 ? "sufficiente"
             :              "insufficiente";
console.log(giudizio);   // "sufficiente"

// ❌ Tre livelli: da evitare — usa if/else if oppure switch
// let r = a ? "a" : b ? "b" : c ? "c" : "d";  ← illeggibile!

// ✅ Equivalente leggibile con if/else if:
let r;
if      (voto >= 9) r = "ottimo";
else if (voto >= 7) r = "buono";
else if (voto >= 6) r = "sufficiente";
else                r = "insufficiente";
console.log(r);          // "buono"

2. L'istruzione switch

Confronta un valore contro una serie di casi precisi. Non esiste in Python fino a Python 3.10 (dove è stato introdotto match/case con semantica diversa). In JS è utile quando hai molte condizioni di uguaglianza sullo stesso valore. 

switch (espressione) {
    case valore1:
        // codice
        break;          // ⚠️ obbligatorio per uscire dal blocco
    case valore2:
        // codice
        break;
    default:            // opzionale: se nessun case corrisponde
        // codice
}
Regola #3 — Senza break si verifica il "fall-through".
Se dimentichi il break, l'esecuzione continua nel case successivo anche se la condizione non corrisponde. Questo causa bug silenziosi difficili da trovare. Metti sempre break (o return se sei dentro una funzione), tranne quando vuoi deliberatamente il fall-through per raggruppare casi.
Regola #4 — switch usa confronto stretto (===).
switch ("1") non corrisponde a case 1: perché i tipi sono diversi. È lo stesso comportamento di ===: nessuna coercizione di tipo. 
// ── Esempio base: giorno della settimana ──────────────────────────────
let giorno = "lunedì";

switch (giorno) {
    case "lunedì":
    case "martedì":
    case "mercoledì":
    case "giovedì":
    case "venerdì":
        console.log("Giorno lavorativo");
        break;                         // ⚠️ obbligatorio
    case "sabato":
    case "domenica":
        console.log("Weekend!");
        break;
    default:
        console.log("Giorno non valido");
}
// stampa: "Giorno lavorativo"

// ── Fall-through VOLONTARIO (casi raggruppati) ─────────────────────────
// Lasciare case senza break per farli "cadere" nel successivo
// è una tecnica intenzionale: i case "lunedì"…"venerdì" sopra ne sono un esempio.

// ── Fall-through ACCIDENTALE (bug) ────────────────────────────────────
let colore = "rosso";
switch (colore) {
    case "rosso":
        console.log("Stop");
        // ❌ manca il break!
    case "giallo":
        console.log("Attenzione");   // ← viene eseguito anche questo! BUG
        break;
    case "verde":
        console.log("Vai");
        break;
}
// stampa: "Stop" e poi "Attenzione"  ← REGOLA #3: senza break cade nel case dopo

// ── REGOLA #4: switch usa === (nessuna coercizione) ────────────────────
switch (1) {
    case "1":               // "1" !== 1 → non corrisponde
        console.log("stringa");
        break;
    case 1:                 // 1 === 1 → corrisponde
        console.log("numero");
        break;
}
// stampa: "numero"

// ── switch con return (dentro una funzione — no break necessario) ──────
function descriviVoto(voto) {
    switch (true) {            // trick: confronta booleani con true
        case voto >= 9: return "Ottimo";
        case voto >= 7: return "Buono";
        case voto >= 6: return "Sufficiente";
        default:        return "Insufficiente";
    }
}
console.log(descriviVoto(9.5));  // "Ottimo"
console.log(descriviVoto(7));    // "Buono"
console.log(descriviVoto(4));    // "Insufficiente"

2.1 switch vs if/else if vs ternario — quando usare cosa

Strumento Usa quando… Evita quando…
? : ternario Hai 2 casi e vuoi un valore da assegnare Hai più di 2 casi o logica complessa
if / else if Le condizioni sono intervalli o espressioni (>=, &&…) Confronti uguaglianza multipla sullo stesso valore
switch Confronti uguaglianza esatta su uno stesso valore con 3+ casi Condizioni con range o espressioni booleane complesse
