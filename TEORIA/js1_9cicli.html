1.9 — Cicli

JavaScript ha cinque tipi di ciclo. Tutti eseguono un blocco di codice ripetutamente, ma differiscono per quando controllano la condizione e su cosa iterano.

CicloUsa quando…Esiste in Python?
forConosci il numero di iterazioniSì (for i in range(n))
whileCondizione booleana genericaSì (identico)
do...whileVuoi eseguire almeno una voltaNo
for...ofIterare sui valori di un array/stringaSì (for x in lista)
for...inIterare sulle chiavi di un oggettoSì (for k in dizionario)

1. for classico

Il for ha tre parti nella testata, separate da ;: inizializzazione, condizione di continuazione, aggiornamento. 

for (inizializzazione; condizione; aggiornamento) {
    // corpo
}
Confronto con Python: 
// JavaScript
for (let i = 0; i < 5; i++) {
    console.log(i);               // 0 1 2 3 4
}

// Python — equivalente
// for i in range(5):
//     print(i)
Regola #1 — Usa let, non var, per il contatore.
Con var il contatore "sfugge" fuori dal blocco for ed è accessibile dopo il ciclo. Con let è confinato all'interno del ciclo — comportamento atteso e sicuro. 
// ── Contare in avanti ─────────────────────────────────────────────────
for (let i = 0; i < 5; i++) {
    console.log(i);               // 0, 1, 2, 3, 4
}

// ── Contare all'indietro ──────────────────────────────────────────────
for (let i = 5; i > 0; i--) {
    console.log(i);               // 5, 4, 3, 2, 1
}

// ── Passo diverso da 1 ────────────────────────────────────────────────
for (let i = 0; i <= 10; i += 2) {
    console.log(i);               // 0, 2, 4, 6, 8, 10
}

// ── Iterare un array con indice ───────────────────────────────────────
let colori = ["rosso", "verde", "blu"];
for (let i = 0; i < colori.length; i++) {
    console.log(i, colori[i]);    // 0 "rosso" | 1 "verde" | 2 "blu"
}

// REGOLA #1 — var sfugge al blocco (evitare!)
for (var j = 0; j < 3; j++) { /* ... */ }
console.log(j);                   // 3 ← j esiste ancora! con let sarebbe ReferenceError

2. while

Controlla la condizione prima di ogni iterazione. Se la condizione è già falsa all'inizio, il corpo non viene mai eseguito. Sintassi quasi identica a Python — cambiano solo le parentesi. 

// ── Esempio base ──────────────────────────────────────────────────────
let i = 0;
while (i < 5) {
    console.log(i);   // 0, 1, 2, 3, 4
    i++;              // ⚠️ senza questo diventa un loop infinito!
}

// ── Condizione mai vera → corpo non eseguito ──────────────────────────
let x = 10;
while (x < 5) {
    console.log("mai eseguito");
}
console.log("dopo il while");   // "dopo il while"  ← il while è stato saltato

// ── Uso tipico: attendere un input valido ─────────────────────────────
// (simulato senza prompt per non bloccare la pagina)
let tentativi = 0;
let numeroSegreto = 7;
let tentativo = 0;          // valore fittizio per far partire il loop

while (tentativo !== numeroSegreto) {
    tentativi++;
    tentativo = tentativi;    // simuliamo un input crescente
}
console.log("Trovato in", tentativi, "tentativi");   // "Trovato in 7 tentativi"
Regola #2 — Ogni ciclo deve avere una via d'uscita garantita.
Assicurati sempre che la condizione del while diventi false ad un certo punto, oppure usa break per uscire. Un loop infinito congela il browser.

3. do...while (non esiste in Python)

Esegue il corpo almeno una volta, poi controlla la condizione. La condizione è in fondo, non all'inizio. 

do {
    // corpo — eseguito ALMENO UNA VOLTA
} while (condizione);   // ← punto e virgola obbligatorio!
Regola #3 — do...while è perfetto quando serve almeno un'esecuzione.
Il caso più comune è chiedere un input all'utente: devi mostrare il prompt almeno una volta prima di poter controllare la risposta. Dichiarare la variabile come let risposta; (senza valore iniziale = undefined) è corretto perché il valore viene assegnato prima che la condizione venga valutata. 
// ── Differenza visiva: while vs do...while ────────────────────────────
let n = 10;

// while: condizione falsa subito → corpo MAI eseguito
while (n < 5) {
    console.log("while:", n);   // non stampato
}

// do...while: corpo eseguito ALMENO UNA VOLTA anche se la condizione è falsa
do {
    console.log("do...while:", n);   // "do...while: 10"  ← eseguito una volta
} while (n < 5);                    // poi la condizione è falsa → si ferma

// ── REGOLA #3: pattern con prompt ────────────────────────────────────
// (spiegazione — non eseguiamo prompt per non bloccare la pagina)
//
// let risposta;           ← undefined: ok, non viene letta prima dell'assegnazione
// do {
//     risposta = prompt("Scrivi 'esci' per uscire:");
//     // ↑ qui risposta riceve un valore PRIMA che la condizione venga valutata
// } while (risposta !== "esci");  ← ora risposta è una stringa, non undefined

// ── Simulazione senza prompt ──────────────────────────────────────────
let risposte = ["no", "ancora no", "esci"];   // input simulati
let risposta;
let idx = 0;

do {
    risposta = risposte[idx++];
    console.log("Input ricevuto:", risposta);
} while (risposta !== "esci");

console.log("Uscito dal loop");
// "Input ricevuto: no"
// "Input ricevuto: ancora no"
// "Input ricevuto: esci"
// "Uscito dal loop"

4. for...of — iterare sui valori

Itera sui valori di qualsiasi oggetto iterabile: array, stringhe, Set, Map. È l'equivalente del for x in lista di Python.

Confronto con Python: 
// JavaScript
let frutti = ["mela", "banana", "arancia"];
for (let frutto of frutti) {
    console.log(frutto);   // "mela", "banana", "arancia"
}

// Python — equivalente
// for frutto in frutti:
//     print(frutto)
// ── Su array ──────────────────────────────────────────────────────────
let numeri = [10, 20, 30];
for (let n of numeri) {
    console.log(n);           // 10, 20, 30
}

// ── Su stringa (itera carattere per carattere) ────────────────────────
for (let ch of "ciao") {
    console.log(ch);          // "c", "i", "a", "o"
}

// ── Con indice: usa entries() ─────────────────────────────────────────
// Se hai bisogno dell'indice oltre al valore, usa .entries()
let colori = ["rosso", "verde", "blu"];
for (let [i, colore] of colori.entries()) {
    console.log(i, colore);   // 0 "rosso" | 1 "verde" | 2 "blu"
}
// Equivalente Python: for i, colore in enumerate(colori)

5. for...in — iterare sulle chiavi di un oggetto

Itera sulle chiavi (proprietà enumerabili) di un oggetto. Il valore si ottiene con oggetto[chiave].

Confronto con Python: 
// JavaScript
let persona = { nome: "Mario", eta: 20, citta: "Roma" };
for (let chiave in persona) {
    console.log(`${chiave}: ${persona[chiave]}`);
}
// "nome: Mario" | "eta: 20" | "citta: Roma"

// Python — equivalente
// for chiave in persona:
//     print(f"{chiave}: {persona[chiave]}")
Regola #4 — Non usare for...in su un array.
Su un array, for...in itera sugli indici come stringhe ("0", "1"…), non sui valori. Può anche includere proprietà aggiunte al prototipo. Per gli array usa sempre for...of o .forEach(). 
// ── for...in su oggetto (uso corretto) ────────────────────────────────
let prodotto = { nome: "Laptop", prezzo: 999, disponibile: true };

for (let chiave in prodotto) {
    console.log(chiave, "→", prodotto[chiave]);
}
// "nome → Laptop"
// "prezzo → 999"
// "disponibile → true"

// ── for...in su array (evitare — REGOLA #4) ───────────────────────────
let arr = ["a", "b", "c"];
for (let i in arr) {
    console.log(typeof i, i);   // "string" "0" | "string" "1" | "string" "2"
}                               // ← gli indici sono stringhe, non numeri!

// ✅ Modo corretto per array: for...of
for (let el of arr) {
    console.log(el);            // "a", "b", "c"
}

6. break e continue

Funzionano esattamente come in Python: break esce dal ciclo, continue salta all'iterazione successiva. 

// ── break e continue insieme ──────────────────────────────────────────
for (let i = 0; i < 10; i++) {
    if (i === 3) continue;   // salta il 3 → va a i=4
    if (i === 7) break;      // esce quando i=7 → non stampa 7,8,9
    console.log(i);            // 0, 1, 2, 4, 5, 6
}

// ── break in un while ─────────────────────────────────────────────────
let somma = 0;
let k = 1;
while (true) {              // loop apparentemente infinito
    somma += k;
    if (somma > 20) break;  // ← la vera condizione di uscita
    k++;
}
console.log("Somma:", somma, "k:", k);   // "Somma: 21 k: 6"

// ── continue per saltare elementi in un array ─────────────────────────
let voti = [7, 4, 9, 3, 8, 5];
console.log("Voti sufficienti:");
for (let voto of voti) {
    if (voto < 6) continue;   // salta i voti insufficienti
    console.log(voto);           // 7, 9, 8
}
