1.5 — Tipi primitivi e typeof

JavaScript, come Python, è un linguaggio a tipizzazione dinamica: non si dichiara il tipo di una variabile — viene inferito automaticamente dal valore assegnato. 

let nome       = "Mario";   // string
let eta        = 20;         // number
let isStudente = true;       // boolean
let voto       = null;       // null (assenza intenzionale di valore)
let indirizzo;               // undefined (dichiarata ma senza valore)

1. I tipi primitivi

JavaScript ha 7 tipi primitivi. I primi cinque sono quelli che userai ogni giorno.

Tipo Esempio Equivalente Python
string "ciao", 'ciao', `ciao` str
number 42, 3.14, NaN, Infinity int, float
boolean true, false True, False (maiuscolo!)
null null None
undefined undefined — (non esiste in Python)
bigint 123n Python gestisce big int nativamente
symbol Symbol("id") — (non esiste in Python)
💡 In Python hai int e float separati. In JavaScript esiste un solo tipo number per tutti i numeri (interi e decimali). Eccezione: bigint per numeri enormi che superano i limiti di number.

2. L'operatore typeof

typeof è un operatore unario che restituisce una stringa con il nome del tipo del valore esaminato. 

console.log(typeof "ciao");      // "string"
console.log(typeof 42);          // "number"
console.log(typeof 3.14);        // "number"  ← interi e decimali sono lo stesso tipo
console.log(typeof true);        // "boolean"
console.log(typeof undefined);   // "undefined"
console.log(typeof null);        // "object"  ← 🐛 BUG STORICO di JavaScript!
console.log(typeof [1, 2]);      // "object"  ← gli array sono oggetti in JS
console.log(typeof {});          // "object"
console.log(typeof function(){}); // "function"
🐛 Bug storico: typeof null restituisce "object" invece di "null". È un errore presente fin dalla prima versione di JavaScript (1995) e non è mai stato corretto per non rompere il codice esistente. Tutti i programmatori JS lo conoscono e lo aggirano.
📌 Come controllare se qualcosa è null: 
let valore = null;

// ❌ NON fare così (typeof non funziona per null):
typeof valore === "null"   // false! restituisce "object"

// ✅ Fare così:
valore === null            // true — confronto diretto

3. null vs undefined

Questo è un concetto nuovo rispetto a Python, dove esiste solo None. In JavaScript ci sono due valori "assenti" con significati diversi. 

let a;              // undefined → "esiste ma non ha ancora un valore"
let b = null;       // null → "ho deciso intenzionalmente che non ha valore"

console.log(a);     // undefined
console.log(b);     // null

console.log(typeof a);  // "undefined"
console.log(typeof b);  // "object"  ← il bug di prima
💡 Analogia:
undefined è come una scatola vuota appena comprata — non hai ancora messo niente dentro.
null è una scatola che hai svuotato di proposito — sai benissimo che è vuota. 
// Confronto == (uguaglianza debole): null e undefined sono "uguali"
console.log(null == undefined);   // true  ← entrambi "assenti"

// Confronto === (uguaglianza stretta): sono tipi diversi
console.log(null === undefined);  // false ← tipi diversi

// In pratica: usa sempre === per confronti precisi (vedi lezione 1.6)

4. Conversioni di tipo automatiche (coercion)

JavaScript converte i tipi automaticamente in certi contesti. Questo può creare risultati sorprendenti, specialmente per chi viene da Python. 

// + con una stringa → concatenazione (la stringa "vince")
console.log("5" + 3);         // "53"   ← non 8!
console.log("ciao" + true);   // "ciaotrue"

// -, *, / → conversione a numero
console.log("5" - 3);         // 2
console.log("5" * 2);         // 10
console.log("10" / 2);        // 5

// boolean → numero (true=1, false=0)
console.log(true + 1);        // 2
console.log(false + 1);       // 1
console.log(true + true);     // 2

// Confronto debole == con tipi misti → risultati sorprendenti
console.log("" == false);     // true  😱
console.log(0 == false);      // true  😱
console.log("0" == false);    // true  😱
In Python "5" + 3 lancia un TypeError — Python non fa conversioni implicite tra tipi incompatibili. JavaScript invece "tenta di aiutarti" e spesso crea bug silenziosi. Per questo è importante usare === (confronto stretto) invece di ==.

5. Conversioni esplicite

Meglio convertire esplicitamente piuttosto che affidarsi alla coercizione automatica. 

// → Number
Number("42");         // 42
Number("3.14");       // 3.14
Number("ciao");       // NaN (Not a Number — non convertibile)
Number(true);         // 1
Number(false);        // 0
Number(null);         // 0
Number(undefined);    // NaN

// parseInt e parseFloat: estraggono numeri da stringhe
parseInt("42px");     // 42  (ignora il resto non numerico)
parseFloat("3.14em"); // 3.14
parseInt("px42");     // NaN (la stringa non inizia con un numero)

// → String
String(42);           // "42"
String(true);         // "true"
String(null);         // "null"
String(undefined);    // "undefined"
(42).toString();      // "42"  (metodo alternativo)

// → Boolean
Boolean(0);           // false
Boolean("");          // false
Boolean(null);        // false
Boolean(undefined);   // false
Boolean(NaN);         // false
Boolean(1);           // true
Boolean("ciao");      // true
Boolean([]);          // true  ← attenzione: array vuoto è truthy!
Valori "falsy" in JavaScript (diventano false in un contesto booleano):

0, "", null, undefined, NaN, false

Tutto il resto è truthy — inclusi [] (array vuoto) e {} (oggetto vuoto)! Questo è diverso da Python, dove [] è False.
