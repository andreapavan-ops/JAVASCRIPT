Ciao JavaScript!

È JavaScript nativo (vanilla JS), nessuna libreria esterna.

Lettura riga per riga:

let output = document.getElementById("output");
document → rappresenta la pagina HTML
.getElementById("output") → cerca nell'HTML un elemento con id="output"
let output = ... → salva quell'elemento in una variabile

output.innerHTML = "";
innerHTML → è il contenuto HTML interno dell'elemento
= "" → lo svuota (cancella tutto ciò che c'è dentro)

In pratica: trova la casella di output nella pagina e la pulisce prima di scrivere nuovi risultati.

Riassunto dubbi:

1. let output = document.getElementById("output")
Non salva il contenuto del div, salva il riferimento (l'indirizzo) all'elemento HTML. Il div può essere vuoto — non importa.

2. output.innerHTML = ""
Svuota il div prima di scrivere. Sembra inutile la prima volta (div già vuoto), ma serve dalla seconda esecuzione in poi per non accumulare risultati vecchi.

3. output.innerHTML += testo + "<br>"
output → sa già qual è il div (collegato al punto 1)
innerHTML → è il contenuto HTML dentro quel div
+= → aggiunge senza sovrascrivere
"<br>" → manda a capo ogni risultato
Senza questa riga il risultato verrebbe calcolato ma non mostrato da nessuna parte.

4. Come fa a sapere dove scrivere?
Perché output è stato "agganciato" al div nella prima riga. Da quel momento output e <div id="output"> sono la stessa cosa in memoria.
